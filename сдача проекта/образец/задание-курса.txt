Ваш итоговый проект
ИИ-система для компании, предоставляющей дома в аренду

Цель проекта

Разработать ИИ-ассистента для компании по аренде домов, который помогает клиентам получать информацию о доступных объектах недвижимости, условиях аренды и стоимости, используя базу знаний объектов и большие языковые модели (LLM).

Система должна уметь:
- отвечать на вопросы клиентов о домах, доступных для аренды
- предоставлять информацию об условиях аренды и характеристиках объектов
- помогать пользователю подобрать подходящий дом
- принимать заявки на аренду
- использовать базу знаний объектов недвижимости для формирования ответов

Проект демонстрирует навыки студента в области:
- промпт-инжиниринга
- разработки AI-ассистентов
- создания RAG-систем
- интеграции LLM через API
- работы с векторными базами данных
- автоматизации процессов через API и No-Code инструменты

Общая информация о проекте

В рамках проекта студент разрабатывает ИИ-ассистента для консультации клиентов по аренде недвижимости.

Ассистент должен использовать базу знаний объектов недвижимости и большие языковые модели для генерации ответов на запросы пользователей.

Система реализуется как RAG-подобный ассистент, который получает информацию из базы данных объектов недвижимости и использует её для формирования точных и релевантных ответов.

Взаимодействие пользователя с системой происходит через Telegram Bot API, а логика обработки запросов и работа с базой знаний может быть реализована с использованием:
- Python
- LangChain
- n8n или Albato

Для поиска информации по объектам недвижимости используется векторная база данных (FAISS).

Система должна позволять пользователю:
- получать информацию о доступных домах
- узнавать характеристики объектов недвижимости
- получать ответы на вопросы об аренде
- отправлять заявку на аренду

Область применения

Описание решаемой задачи

Компании, занимающиеся арендой домов, получают большое количество однотипных запросов от клиентов, связанных с:
- характеристиками объектов недвижимости
- стоимостью аренды
- условиями проживания
- доступностью домов

Ответы на такие вопросы часто требуют обращения к базе данных объектов.

Использование ИИ-ассистента позволяет:
- автоматизировать консультации клиентов
- ускорить процесс получения информации
- снизить нагрузку на сотрудников
- улучшить качество клиентского сервиса

Пользователи системы

Основными пользователями системы являются:
Клиенты компании, которые ищут дома для аренды и хотят получить информацию об объектах недвижимости.

Дополнительно системой могут пользоваться менеджеры компании, которые получают заявки на аренду.

Ограничения и допущения
- Система работает через Telegram-бота
- Ответы на запросы формируются с использованием LLM через API
- Информация об объектах недвижимости хранится в базе знаний
- Поиск информации по объектам реализуется через векторную базу данных

Функциональные требования

ИИ-ассистент
Система должна быть реализована в виде ассистента, который отвечает на вопросы пользователей и помогает подобрать дом для аренды.

Основные функции ассистента:

Консультация пользователей
Ассистент должен уметь:
- отвечать на вопросы о доступных домах
- предоставлять информацию о характеристиках объектов
- объяснять условия аренды
- предоставлять информацию о стоимости

Ответы формируются на основе базы знаний объектов недвижимости.

Подбор объектов недвижимости
Ассистент должен помогать пользователю:
- найти подходящий дом для аренды
- подобрать объект по характеристикам (например, расположение, цена, тип дома)
- получать рекомендации по объектам недвижимости

Прием заявок на аренду
Пользователь должен иметь возможность:
- отправить заявку на аренду дома
- указать интересующий объект

После отправки заявки система должна зафиксировать данные запроса.

Интеграции

1. LLM и API генерации текста
Используются для:
- генерации ответов пользователям
- обработки запросов клиентов
- формирования рекомендаций по объектам недвижимости

2. Векторная база данных
Используется для:
- хранения базы знаний объектов недвижимости
- поиска релевантной информации для ответов ассистента

В проекте может использоваться FAISS.

3. Telegram Bot API
Используется для:
- взаимодействия пользователей с системой
- отправки и получения сообщений
- приема заявок от клиентов

4. Автоматизация сценариев
В проекте может использоваться один из инструментов:
- Python
- LangChain
- n8n
- Albato

Эти инструменты используются для реализации логики обработки запросов и интеграции компонентов системы.

Общая архитектура системы

Типовая архитектура проекта
Интерфейс пользователя (Telegram-бот)
↓
Логика обработки запросов (Python / LangChain / сценарии автоматизации (n8n или Albato))
↓
RAG-механизм (поиск информации по базе знаний)
↓
Векторная база данных (FAISS)
↓
AI-модель (LLM через API)
↓
Ответ пользователю (Telegram Bot API)

Требования к промптам
Студент должен разработать набор промптов, обеспечивающих:
- корректную работу ассистента
- генерацию ответов на основе данных из базы знаний
- помощь пользователю в подборе объектов недвижимости

Промпты должны:
- задавать роль консультанта по аренде недвижимости
- учитывать информацию из базы знаний
- формировать структурированные ответы для пользователей

Методы тестирования

1. Тестирование ассистента
Необходимо проверить:
- корректность ответов на вопросы пользователей
- соответствие ответов данным базы знаний
- логичность рекомендаций по объектам недвижимости

2. Тестирование поиска по базе знаний
Проверяется:
- корректность поиска информации
- релевантность найденных объектов
- корректная передача данных в LLM

3. Тестирование Telegram-бота
Проверяется:
- корректность обработки пользовательских сообщений
- отправка ответов пользователю
- прием заявок на аренду

4. Разработка Telegram-бота
- создание бота
- настройка взаимодействия пользователей с ассистентом
